# Player character asset contract

`player-placeholder.glb` is a project-owned, reproducible mannequin generated by
`npm run character:build`. It validates the runtime character pipeline without making game startup depend on a
third-party download.

A replacement GLB must:

- face positive Z with its feet at the origin; runtime normalises the visible height to 1.8 world units;
- contain the humanoid nodes listed in `RiggedPlayerVisual.ts` (standard Mixamo and Rigify aliases are accepted);
- contain in-place `idle`, `walk`, and `sprint` clips; the canonical optional clips are `aim`, `aim_forward`,
  `aim_back`, `aim_left`, `aim_right`, `fire`, `punch_left`, `punch_right`, `jump`, `fall`, `land`, `death`, and `ride`;
- stay below 60,000 triangles, four materials, 2048px per texture, and 10 MB transferred;
- include no unlicensed third-party likeness, logo, clothing design, mesh, texture, or animation.

FBX and Blender files are source/interchange formats only. The game ships and loads GLB.
